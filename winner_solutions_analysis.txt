 1ST PLACE — HARRISON HORAN ("Harry")   Brier: 0.1097454
================================================================================

PHILOSOPHY
----------
  "Let the selection committee do the bulk of the work (trust seeding),
   then add features for what seeding doesn't capture."
  - Trusted seed as the backbone, not an enemy
  - Treated men's and women's as separate entities
  - Worked backwards from Brier score ("finish in the money or bust")
  - Minimal features, maximum care

PIPELINE
--------
  1. Start from raddar's 2025 kaggle model + Rob Mulla's 2023 tutorial
  2. Compute efficiency metrics per game:
         Poss = FGA - OR + TO + 0.475 * FTA
         OffEff = (Score / Poss) * 70
         DefEff = (OppScore / OppPoss) * 70
         NetEff = OffEff - DefEff
  3. Manual injury adjustments for men (women skipped):
         status_weights = {
             "Out For Season": 1.0,
             "Out":            0.75,
             "Game Time Decision": 0.5,
         }
     Sources:
         - Injury status: rotowire.com
         - Player ratings: evanmiya.com (manual data entry!)
  4. Season-average Net Efficiency with 2% trim mean (robust to blowouts)
  5. Subtract injury deductions from Net Rating
  6. Build custom "harry_Rating":
         harry_Rating = NetEff
                      * (1 + opp_quality_MinMax)     # SOS scaler
                      * power_conf_MinMax            # power conference bonus
                      * top12_MinMax                 # AP Poll week 6 top-12

FEATURES (only 3-4 per gender!)
-------------------------------
  Men's:
    - seed_diff                    r = -0.5937
    - opp_qlty_pts_won_diff        r =  0.4603  (sum of opp-quality pts on wins)
    - harry_diff                   r =  0.5971  (custom rating differential)

  Women's:
    - seed_diff                    r = -0.7560
    - opp_qlty_pts_won_diff        r =  0.6437
    - harry_diff                   r =  0.7743
    - avg_blk_diff                 r =  0.4373

MODEL
-----
  Single XGBoost Regressor (target = Win, binary 0/1):
    n_estimators      = 4000
    learning_rate     = 0.003
    early_stopping    = 100
    max_depth         = 2   (!)   <-- very shallow; prevents overfitting
    min_child_weight  = 5
    subsample         = 0.7
    colsample_bytree  = 0.8
    reg_alpha         = 0.1
    reg_lambda        = 1.0

VALIDATION
----------
  Leave-One-Season-Out CV (2003-2025)
  CV Brier: 0.1620 raw -> 0.1590 calibrated

CALIBRATION
-----------
  Isotonic Regression post-hoc (one per gender):
    men_calibrator.fit(oof_preds, oof_labels)
  Impact:
    Men's CV Brier:   0.1850 -> 0.1822
    Women's CV Brier: 0.1390 -> 0.1357

POST-PROCESSING (the winning move)
----------------------------------
  "Kill your darlings" / edge sharpening:
    - Round predictions >= 0.97 up to 1.0
    - Round predictions <= 0.03 down to 0.0
  - Applied to 28 first-round games in practice
  - Negative-EV gamble: P(at least one miss across 35 games) = 65.6%
  - But improves score by ~0.00011 when it works out
  - Author notes: lucky to land on 0.97/0.03 thresholds; 2nd place used 0.02

WHAT HARRY DID WELL

  + Manual injury research (+~0.0041 Brier; highest single-intervention gain)
  + Treated men/women as separate problems

 2ND PLACE — BRENDAN CARLIN                      Brier: 0.1149886
================================================================================

PHILOSOPHY
----------
  Ensemble two complementary learners (LR for calibration, XGB for
  non-linearity), use many small features framed as differences, ruthlessly
  optimize Brier score with probability clipping.

PIPELINE
--------
  1. Source data: Kaggle competition files only (no external data)
  2. Build team-season stats from M/W RegularSeason Detailed/Compact Results
  3. Compute efficiency per 100 possessions:
         Poss    = FGA - OR + TO + 0.475 * FTA
         OffEff  = TotalPts    / Poss * 100
         DefEff  = TotalOppPts / OppPoss * 100
         NetEff  = OffEff - DefEff
  4. Compute Four Factors (Dean Oliver):
         EFGPct = (FGM + 0.5 * FGM3) / FGA
         TORate = TO / Poss
         ORPct  = OR / (OR + OppDR)
         FTRate = FTA / FGA
  5. Compute margin-of-victory Elo ratings:
         K = 20
         home_adv = 100
         mean_reversion = 0.75 (revert to 1500 each season)
         mov_mult = log(max(mov,1)+1) * (2.2 / ((w_elo-l_elo)*0.001 + 2.2))
  6. Aggregate Massey ordinals (last 2 weeks of regular season):
         MasseyMean, MasseyMedian, MasseyMin
     Women's teams: fixed default of 200 (Massey is men-only in comp data)
  7. Build Strength of Schedule = avg(opp win pct)
  8. Build Recent Form = last-10-game win pct
  9. Frame ALL features as Team1 - Team2 where Team1.id < Team2.id
 10. Combine men's + women's into a single training set

FEATURES (35 total, ALL DIFFERENCES)
------------------------------------
  Basic season stats:   WinPct, PointDiff, AvgPts, AvgOppPts
  Advanced efficiency:  OffEff, DefEff, NetEff
  Four Factors:         EFGPct, TORate, ORPct, FTRate
  Shooting splits:      FGPct, FG3Pct, FTPct, OppFGPct
  Per-game stats:       Ast, TO, Stl, Blk, OR, DR, Tempo
  Opponent factors:     OppEFGPct, OppTORate, DRPct, OppFTRate
  Ratings:              EloDiff, SeedDiff, Seed1, Seed2
  Massey (men only):    MasseyMean, MasseyMedian, MasseyMin diffs
  Other:                SOSDiff, RecentFormDiff

MODELS
------
  Logistic Regression (on StandardScaler-normalized features):
    C = 1.0
    max_iter = 1000
    solver = lbfgs

  XGBoost Classifier (on RAW unscaled features):
    n_estimators      = 300
    max_depth         = 4   (!)   <-- deeper than Harry's 2
    learning_rate     = 0.05
    subsample         = 0.8
    colsample_bytree  = 0.8
    min_child_weight  = 3
    reg_alpha         = 0.1
    reg_lambda        = 1.0

  BLEND:
    blended = 0.4 * lr_preds + 0.6 * xgb_preds
    blended = np.clip(blended, 0.02, 0.98)     # Brier-aware clipping

CV BRIER SCORES (10-fold on neg_brier_score)
--------------------------------------------
                          Men's            Women's
  Logistic Regression  0.1903 ± 0.0159   0.1425 ± 0.0111
  XGBoost              0.2009 ± 0.0170   0.1552 ± 0.0180
  Blended (40/60)      0.1925 ± 0.0162   0.1480 ± 0.0151

  LR outperformed XGBoost alone — the blend wins because XGB adds
  non-linear interactions that complement LR's calibrated linearity.

FINAL COMPETITION BRIER: 0.1149886
  (much better than CV ~0.15-0.19 because 2026 was a chalky year)

WHAT BRENDAN DID WELL
---------------------
  + Difference-vector framing generalizes across eras and genders
  + Combined men + women training set (doubles sample; works because
    relative-strength dynamics are gender-agnostic)
  + Massey ordinals aggregation (strongest non-seed men's feature)
  + MOV-weighted Elo with autocorrelation correction to prevent runaway
  + Scale-aware pipeline: LR gets scaled features, XGB gets raw
    (LR coefficients are scale-sensitive; tree-based models aren't)
  + Probability clipping [0.02, 0.98] - direct Brier optimization
  + Tuned max_depth=4 specifically (6 and 8 overfit on ~1000 games)
  + Fully reproducible (random_state=42, deterministic, bit-for-bit)
  + 10-fold CV on neg_brier_score (tunes against competition metric)
  + No external data needed (simpler, more auditable)

WHAT BRENDAN DID BADLY / LIMITATIONS
-------------------------------------
  - No Massey ordinals for women (competition data gap; fell back to
    constant 200 for all women; clear information loss)
  - 60/40 blend ratio was empirical, not grid-searched
    (should have used nested CV over blend weights)
  - No upset-specific seed-matchup features (12v5 ~35%, 11v6 ~37% are
    stable signals; currently captured implicitly only)
  - Single combined model: training M+W together sacrifices
    tournament-specific tuning
  - No isotonic/Platt calibration layer after blending — relies on LR's
    implicit calibration
  - No injury adjustments (Harry got +0.004 from these)
  - No recent form weighting beyond a single last-10-games feature
  - Men's-women's feature-count asymmetry (35 men vs 32 women) could
    bias the combined model toward men's patterns

POSSIBLE IMPROVEMENTS (Brendan's own list)
------------------------------------------
  1. Women's composite rankings (ESPN BPI, Her Hoop Stats, NCAA NET) to
     close the Massey-ordinal gap
  2. Post-blend isotonic regression calibration
  3. Proper grid search over blend weights with nested CV
  4. Seed x Massey interaction features for dangerous mid-seed teams
  5. Separate men's and women's models with post-hoc calibrated blend
  6. Tournament-trajectory features (does bracket path affect later rounds?)

 3RD PLACE — KEVIN E R MILLE                     Brier: 0.1160374
================================================================================

PHILOSOPHY
----------
  "For small-data prediction tasks (~650 rows), simpler models with careful
   feature selection beat complex models every time."
  - Logistic Regression (not XGBoost) on small tournament data
  - Prediction markets provide orthogonal information a pre-tournament
    statistical model cannot capture
  - Aggressive feature pruning: 33 -> 23 (men), 19 -> 11 (women)
  - Separate men's and women's models
  - 52 rounds of iteration over 4 weeks (Claude Code as pair programmer)

PIPELINE
--------
  1. Data sources: competition data + Barttorvik + Massey + KenPom +
     EvanMiya + AP Poll + Coach Results + WNCAA NET + ESPN BPI +
     Vegas moneylines + Kalshi prediction markets
  2. Build team-season stats with Four Factors from M/W detailed results
  3. Compute carry-over Elo:
         K = 32, home_adv = 100 rating points
         mov_mult = ln(|margin|+1) * 2.2 / (0.001*|margin| + 2.2)
         Season carryover: 0.75 * end_elo + 0.25 * 1500
  4. Compute custom rating systems:
         Colley Matrix: solve (2I + C) r = 1 + (w - l)/2
         SRS: iterative rating_i = avg_margin_i + mean(opp_ratings)
         GLM Quality (Raddar): MLE team strength from game graph
  5. Aggregate Massey: mean of latest rankings from all ~28 systems with
     full coverage (2015-2025 excl. 2020). Dynamic system selection beat
     hand-picked subsets by 0.00246 Brier.
  6. Build feature interactions:
         SeedNum x massey_rank, ncsos x massey_rank,
         close_win_pct x pt_diff, srs x ap_rank
  7. Backward elimination on features (one feature at a time; keep if CV
     improves).
  8. Train LR on differential features (Team1 - Team2).
  9. Apply tiered market blend as post-processing on 2026 predictions.

FEATURES
--------
  Men's (25 features):
     1.  SeedNum                    13. ap_rank
     2.  win_pct                    14. em_change (preseason->end EM)
     3.  adjoe                      15. glm_quality (Raddar MLE)
     4.  adjde                      16. SeedNum x pt_diff
     5.  WAB                        17. massey_rank x barthag
     6.  sos                        18. close_win_pct x pt_diff
     7.  adjt                       19. colley_rank
     8.  efg_pct                    20. ncsos
     9.  tov_pct                    21. SeedNum x massey
    10.  oreb_pct                   22. ncsos x massey
    11.  oreb_pct_d                 23. srs
    12.  fg3_pct                    24. srs x ap_rank
                                    25. coach_pase

  Women's (11 features, fewer because women's game is more chalky):
     1.  SeedNum                     7. net_x_elo
     2.  win_pct                     8. elo_x_colley
     3.  oreb_pct_d                  9. srs
     4.  blk_pct                    10. last_n_pt_diff_x_elo_x_colley
     5.  last_n_win_pct             11. last_n_pt_diff_x_colley_rank
     6.  elo_slope

MODEL
-----
  Pipeline:
    SimpleImputer(strategy="median")
    -> StandardScaler()
    -> LogisticRegression(solver="lbfgs", max_iter=1000)

  Hyperparameters:
    Men's:   C=100, clip predictions to [0.03, 0.97]
    Women's: C=0.15, clip predictions to [0.005, 0.995]

  Rationale for C asymmetry:
    - Men: 25 well-conditioned features benefit from more capacity
    - Women: 11 features on ~646 rows needs noise suppression

VALIDATION
----------
  Leave-One-Season-Out via GroupKFold(n_splits=10) grouped by season
  Training: 2015-2025 excluding 2020 (no COVID tournament)
  ~669 men's games, ~646 women's games
  Final LOSO Brier: Men's 0.11199, Women's 0.13506, Combined 0.12352

TIERED MARKET BLEND (the 3rd place secret sauce)
------------------------------------------------
  Tier 1 — Round 1 game-specific (highest confidence):
    Market consensus = 0.6 * Vegas_no_vig + 0.4 * ESPN_BPI
    Vegas no-vig: P_fav = |ML| / (|ML|+100)
                  P_dog = 100 / (ML+100)
                  then normalize so probs sum to 1
    Men's R1:   0.10 * LR + 0.90 * market_consensus
    Women's R1: 0.25 * LR + 0.75 * market_consensus
    Coverage: 30 men's R1 + 28 women's R1 games with BPI+Vegas

  Tier 2 — Non-R1 with BPI championship odds (Bradley-Terry):
    P(A beats B) = strength_A / (strength_A + strength_B)
    Men's:   0.75 * LR + 0.25 * BPI_BT
    Women's: 0.85 * LR + 0.15 * BPI_BT

  Tier 3 — Kalshi futures (Bradley-Terry from championship odds):
    Men's:   0.85 * LR + 0.15 * Kalshi_BT
    Women's: 0.85 * LR + 0.15 * Kalshi_BT

  Tier 4 — Pure LR fallback (no market data):
    Used when no BPI or Kalshi odds exist for the matchup

  Rationale: Markets aggregate millions of dollars of analysis including
  real-time injury/lineup info no pre-tournament model can capture.
  Trust markets most where they're most informative.

================================================================================
 CROSS-CUTTING LESSONS FROM ALL THREE WINNERS
================================================================================

FEATURE PHILOSOPHY
  - Seed differential is ALWAYS the single strongest feature
  - Net efficiency per 100 possessions >> per-game averages
  - Massey ordinals (consensus of 28-200 systems) is stronger than any one
    (Kevin: using all eligible systems beats hand-picked subsets)
  - Fewer features beats more features on small tournament datasets
    (Harry: 3-4 features; Kevin: pruned 33 -> 23; Brendan: 35)
  - Differences (Team1 - Team2) generalize better than absolutes
  - Backward elimination (one-feature-at-a-time) can yield the single
    biggest CV improvement of a project (-0.007 Brier for Kevin in one
    round)
  - Carry-over Elo (season mean reversion) > fresh-start Elo
  - Feature interactions (SeedNum x Massey, etc.) capture non-linearity
    that LR can't model on its own

MODEL PHILOSOPHY
  - Shallow trees (depth 2-4). Tournament data is ~1000 games; deeper
    trees overfit.
  - LOGISTIC REGRESSION on small data (~650 rows) can beat tuned
    XGBoost for Stage 2 generalization. XGB wins Stage 1 by memorizing.
    (Kevin: LR 0.124 vs XGB 0.157 on CV)
  - Small learning rates with many boosting rounds if using trees
  - Strong L1/L2 regularization always
  - Ensembles win only if base learners are TRULY complementary:
      * Brendan's LR+XGB blend worked (60/40, scaled vs raw features)
      * Kevin's LR+XGB blend hurt CV and LB
    Don't blend fundamentally different model types on small data
    unless you have strong reason to believe they're orthogonal.
  - Separate men's and women's models usually win (Harry, Kevin).
    Brendan's combined approach also worked, with difference vectors.

BRIER-SCORE AWARENESS
  - Three distinct clipping strategies in the top three:
      * Harry:   round TO extremes (0.97 -> 1.0)        — aggressive
      * Brendan: clip AWAY from extremes (max 0.98)      — conservative
      * Kevin:   clip AWAY asymmetric ([0.03,0.97] men,  — model-tuned
                 [0.005,0.995] women)
    All three work; different risk profiles.
  - Always optimize against neg_brier_score in CV, not AUC/accuracy
  - Isotonic calibration post-hoc is worth ~0.003 Brier (Harry)
    ...but Kevin's attempt at isotonic FAILED because OOF distributions
    differed from full-model predictions. Watch this failure mode.
  - Injury adjustments worth ~0.004 Brier (Harry — biggest single
    manual gain). Kevin didn't do this; could have improved further.
  - Prediction market blending (Vegas, BPI, Kalshi) can be very
    powerful for Round 1 games but is NOT CV-validatable without
    historical market data — a judgment call, not a science.

CV PHILOSOPHY
  - Leave-One-Season-Out > K-fold random (Harry, Kevin); Brendan used
    10-fold and still placed 2nd.
  - GroupKFold grouped by season ~ LOSO for small season count
  - CV Brier is a CEILING estimate; actual tournament Brier can be
    much better (chalky year like 2026) or much worse (upset year)
  - CV-LB divergence (Stage 1 vs Stage 2) is real: Kevin explicitly
    noted XGB wins Stage 1 by memorization, LR wins Stage 2 by
    generalization. Trust LOSO CV over LB for Stage 2.

DATA SOURCE PHILOSOPHY
  - All three winners used competition game-level data as foundation.
  - Harry and Kevin added substantial EXTERNAL data (Barttorvik,
    AP Poll, coach records, BPI, Vegas, Kalshi, EvanMiya)
  - Brendan used only competition data and still placed 2nd with 35
    careful features — shows external data is helpful but not required
  - Manual external data (Harry's injury player ratings; Kevin's
    market scraping) is the ceiling but hurts reproducibility

    - SOS = avg(opp win pct)
    - RecentForm = last-10-game win pct
    - MasseyMean, MasseyMedian, MasseyMin (last 2 weeks)

  Stage 3: Matchup Feature Building

    - Feature interactions: SeedNum x Massey, etc.

  Stage 4: Modeling

    - XGBoost on raw features (shallow, depth=4)

      SampleSubmissionStage2.csv
    - Default to 0.5 for missing team profiles
